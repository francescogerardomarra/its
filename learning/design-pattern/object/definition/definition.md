# Definition
**Design patterns have the following characteristics:**
- **they are** design-level **solutions for recurring problems** that computer engineers often encounter;
- it's not code;
- i repeat, ❌ NOT CODE;
- it's like a description of how to tackle these problems and design a solution.

The use of these patterns is considered a good practice, as the design
of the solution has been thoroughly tested and proven, resulting in greater
readability of the final code.

Design patterns are created quite often for and used by object-oriented programming (OOP) languages, 
such as Java.

There are approximately 26 design patterns currently known.

**These 26 can be classified into 3 types:**
1. **Creational**:
    - designed to create an instance of a class;
    - they can be either class creation patterns or object creation patterns.
2. **Structural**:
    - designed in relation to the structure and composition of a class;
    - the main goal of most of these patterns is to increase the functionality of the involved class
      (or classes) without changing its composition too much.
3. **Behavioral**:
    - designed depending on how a class communicates with others.
